# 殷墟小卜官项目迁移与构建说明

## 项目信息

- 引擎：Cocos Creator 3.8.8
- 主要开发语言：TypeScript
- Android 包名：`com.yinxu.oracle`
- 当前 Android 架构：`arm64-v8a`
- 主场景：`assets/scene.scene`
- 核心游戏逻辑：`assets/scripts/YinXuCity.ts`

## 源代码包包含内容

- `assets`：场景、脚本、贴图与其他游戏资源
- `extensions`：项目本地扩展
- `native`：Android 原生工程自定义源码
- `profiles`、`settings`：Cocos 项目与构建配置
- `tools`：项目辅助工具
- `package.json`、`tsconfig.json`、`.creator` 等项目文件

`build`、`library`、`temp`、`tmp` 属于编译产物或可重新生成的缓存，没有放入迁移包；APK 也没有放入本包。

## 在另一台电脑打开

1. 安装 Cocos Dashboard 与 Cocos Creator 3.8.8。
2. 将压缩包解压到纯英文路径，例如 `D:\Projects\RPG`。不要放在带中文或特殊字符的目录中，以免 Android Gradle 在 Windows 下出现路径编码问题。
3. 在 Cocos Dashboard 中选择“导入”，定位到解压后的 `RPG` 文件夹。
4. 首次打开会重新生成 `library`、`temp`、`tmp`，等待资源导入完成后再预览或构建。

## Android 构建环境

- JDK：17
- Android SDK Platform：36
- Android Build Tools：36.0.0
- Android NDK：23.2.8568313（r23c）
- CMake：3.22.1

在 Cocos Creator 的“偏好设置 → 程序管理器”中配置 Java Home、Android SDK、Android NDK 和 CMake 路径，然后通过“项目 → 构建 → Android”生成原生工程并执行 Make。

当前测试包使用 Cocos 调试证书。正式上架前必须创建并妥善保存独立的发布签名证书，并在 Android 构建设置中替换调试证书。
